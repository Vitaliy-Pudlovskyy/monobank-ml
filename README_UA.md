# 💳 Monobank ML — Аналіз та Передбачення Витрат

Навчальний проект машинного навчання на основі особистої банківської виписки Монобанку за 2025 рік.

Два рівні реалізації:
- **З нуля** — тільки Python, NumPy, Pandas (без ML фреймворків)
- **Бібліотечний** — sklearn, XGBoost, LightGBM, CatBoost

---

## 🎯 Задачі

| Задача | Тип |
|--------|-----|
| Передбачити категорію витрат на день (0-200 / 200-600 / 600+ UAH) | Класифікація |
| Передбачити суму витрат на день | Регресія |

## 🔬 Архітектури ручних реалізацій

**Нейронна мережа (класифікація):** `6 → 10 → 6 → 3` з ReLU + Softmax

**Нейронна мережа (регресія):** `6 → 6 → 3 → 1` з ReLU

**Дерево рішень:** критерій Джині / MSE, рекурсивне розбиття, max_depth=3

**Random Forest:** bagging + випадкові підвибірки ознак, n=6, голосування більшістю

**Gradient Boosting:** послідовні дерева на залишках `F_m = F_{m-1} + η·h_m`

**XGBoost:** GB + гессіан `hᵢ = pᵢ(1-pᵢ)` + L2/γ регуляризація + оптимальний лист `w* = -ΣG/(ΣH+λ)`

**LightGBM:** XGBoost + leaf-wise побудова дерев + GOSS sampling

**CatBoost:** Ordered Target Encoding категоріальних ознак + Symmetric Trees з cumsum оптимізацією

## 💡 Головні висновки

**Стеля датасету — ~78%.** Три принципово різні алгоритми (GB, LightGBM, CatBoost) з різними параметрами дають однаковий результат 71.2%. Це сигнал що проблема в даних, а не в моделі.

**Регресія — практично нерозв'язна задача на цих даних.**
RMSE 1272 UAH при середньому 851 UAH і std 1535 UAH — модель не краща за наївний baseline. Витрати надто хаотичні.

**Прості моделі виграють на малих даних.**
Random Forest (78%) і Дерево рішень (76%) б'ють XGBoost (74.6%). Складність не компенсує брак даних.

**Параметри вирішують все.**
XGBoost без регуляризації: train 95.7% → test 62.7%. З λ=2.0, γ=0.5: train 75.5% → test 74.6%. Різниця — два числа.

**Ручна ≈ бібліотечна при однакових параметрах.**
При тюнінгу різниця 1-3% — деталі реалізації, не фундаментальні відмінності.
